# Body Quest — game data (JSON)

All content is stored in **`.json` files**, split by purpose. Generated `*-data.js` bundles are built for the browser — do not edit those by hand.

## Question content (edit these)

| File | Purpose |
|------|---------|
| `level1-questions.json` | Level 1 — Organ functions map matching clues |
| `level2-questions.json` | Level 2 — Disease map matching clues |
| `level3-questions.json` | Level 3 — Healing food map matching clues |
| `level4-questions.json` | Level 4 — Symptom diagnosis map matching clues |

After editing, run:

```bash
cd templates/body_quest_templates
node scripts/sync-organs-qa.mjs
```

## Shared config (edit these)

| File | Purpose |
|------|---------|
| `patient.json` | Patient name, age, generic symptoms |
| `organs-core.json` | Organ id, name, icon, disease, nutrient, color, image |
| `organ-hotspots.json` | Body-map hotspot positions (desktop + mobile) |
| `levels-config.json` | Level titles, modes, organ order |

After editing, run:

```bash
cd templates/body_quest_templates
node scripts/sync-organs-shared.mjs
```

## Generated files (do not edit)

| File | Generated by |
|------|----------------|
| `organs-qa-data.js` | `sync-organs-qa.mjs` |
| `organs-shared-data.js` | `sync-organs-shared.mjs` |

## Loaders (logic only)

| File | Purpose |
|------|---------|
| `organs-shared.js` | Applies `organs-shared-data.js` to globals |
| `organs-qa-loader.js` | Loads Q&A into `ORGANS_LEVEL1/2/3` |
| `map-click-challenges.js` | Builds map-click prompt pools |

Script load order in `index.html` matters — see the bottom of that file.
